# Plan: Epic 23.1 - Answer Quality JUnit Report Integration

## Objective

Generate `answer_quality_eval_junit.xml` from the existing answer quality evals and
wire it into corpus maintenance, GitHub Actions artifacts, and the regression
dashboard.

## Scope

- Add a Makefile target for local JUnit generation.
- Add an answer quality eval step to corpus maintenance when `run_evals=true`.
- Extend the regression dashboard JUnit parser with answer quality counters.
- Update GitHub Actions summary/artifact behavior.
- Add dashboard unit tests for clean, failure, error, skipped, and missing JUnit.
- Update docs, EVALS, README, ROADMAP, and Obsidian.

## Out of Scope

- No changes to answer quality metrics or golden cases.
- No changes to RAG retrieval, Action Brief, scoring, diagnosis, or recommendation.
- No LLM judge and no new dependencies.

## Implementation

1. Generate local XML with:
   `python -m pytest tests/evals/test_answer_quality_golden.py --junit-xml=data/regression_reports/answer_quality_eval_junit.xml`.
2. Add `answer-quality-junit` Makefile target that creates `data/regression_reports/`
   and writes the XML there.
3. Add `answer_quality_evals` to `scripts/run_corpus_maintenance.py` after golden evals.
4. Parse answer quality JUnit in `scripts/build_regression_dashboard.py`, extracting
   tests, failures, errors, skipped, failed cases, summarized failure messages, and
   PASS/WARN/FAIL status.
5. Surface answer quality metrics in dashboard Markdown/JSON and GitHub Job Summary.
6. Keep missing XML controlled: dashboard does not crash; status is WARN.

## Tests

- `pytest tests/unit/test_regression_dashboard.py`
- `pytest tests/evals/test_answer_quality_golden.py`
- `pytest`
- `ruff check .`
- `black --check .`
- `mypy src`

## Definition of Done

- XML is generated locally and during corpus maintenance eval runs.
- Dashboard consumes the XML and fails on answer quality failures/errors.
- Workflow publishes the XML through the maintenance artifact.
- Job Summary shows answer quality status and counters.
- Docs and Obsidian are updated.
