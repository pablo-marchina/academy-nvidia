# Plan: Epic 26.2 - Workspace Output Validation Gate

## Objective

Add a workspace Output Validation Gate that requires generated outputs to be
validated for contract, format, scope, evidence/uncertainty preservation, and
operational checks before a task is marked complete.

## Scope

- Add `Workspace Output Validation Gate` to `AGENTS.md`.
- Create focused validators under `src/validation/`.
- Add unit tests and small example fixtures.
- Add Makefile targets for output validation.
- Update README, ROADMAP, EVALS, docs, and Obsidian.

## Out of Scope

- No scoring changes.
- No retrieval changes.
- No recommendation changes.
- No Action Brief generation changes.
- No API/UI behavior changes beyond validating outputs.
- No new dependency.
- No mandatory LLM judge.
- No heavy process for trivial hotfixes.

## Proposed Implementation

1. Add validation result schemas with `PASS`, `WARN`, and `FAIL`.
2. Implement validators for Action Brief JSON, Markdown, dashboard JSON, and API responses.
3. Reuse existing Pydantic schemas and NVIDIA mapping matrix.
4. Add example payloads in `examples/validation/`.
5. Add focused unit tests for positive and negative cases.
6. Add Makefile targets.
7. Update workspace docs and Obsidian backfill.

## Risks

| Risk | Mitigation |
|------|------------|
| Duplicating Answer Quality Eval | Keep this gate structural/contractual and reuse existing schemas. |
| Gate becomes bureaucracy | Exempt trivial hotfixes from heavy process. |
| Markdown validation too strict | Only require headings, non-empty critical sections, and placeholder warnings. |
| NVIDIA mapping drifts | Reuse existing mapping function instead of copying matrix values. |
