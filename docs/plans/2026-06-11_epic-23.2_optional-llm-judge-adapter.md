# Plan: Epic 23.2 - Optional LLM Judge Adapter for Answer Quality

## Objective

Add an optional, experimental LLM judge interface for semantic review of RAG/Action
Brief answer quality without changing deterministic quality gates or CI behavior.

## Scope

- Add LLM judge schemas, prompt builders, adapter base class, and offline null provider.
- Add a manual script that writes JSON and Markdown reports under
  `data/regression_reports/`.
- Add an optional regression dashboard section when the report exists.
- Add tests proving the provider, prompts, script, and dashboard behavior are offline
  and non-blocking.
- Update README, EVALS, ROADMAP, Answer Quality docs, Makefile, `.env.example`, and
  Obsidian notes.

## Out of Scope

- No real provider integration.
- No external API calls or required API keys.
- No changes to deterministic Answer Quality gates or JUnit XML.
- No changes to scoring, diagnosis, recommendation, retrieval, `recommended_motion`,
  or Action Brief logic.
- No new dependencies.

## Implementation

1. Create `src/evaluation/llm_judge_schemas.py` with `LLMJudgeInput`,
   `LLMJudgeScore`, `LLMJudgeResult`, `LLMJudgeRunReport`, and
   `LLMJudgeProviderConfig`.
2. Create `src/evaluation/llm_judge_prompts.py` with six rubric prompts and a full
   prompt builder that includes context, answer, evidence, RAG contexts, gaps,
   recommendations, uncertainty, and deterministic metrics.
3. Create `src/evaluation/llm_judge_adapter.py` with `BaseLLMJudgeProvider`,
   `NullLLMJudgeProvider`, and `run_llm_judge_for_cases`.
4. Create `scripts/run_answer_quality_llm_judge.py` as a manual runner using the null
   provider by default and writing JSON/Markdown reports.
5. Extend `scripts/build_regression_dashboard.py` to read
   `answer_quality_llm_judge_report.json` if present and render an informational
   Optional LLM Judge section without changing dashboard status.
6. Add `make answer-quality-llm-judge` as an explicit optional/experimental target.

## Tests

- `tests/unit/test_llm_judge_adapter.py`
- Existing `tests/unit/test_regression_dashboard.py` extended for report present and
  absent behavior.
- Full validation: `pytest`, `ruff check .`, `black --check .`, `mypy src`.

## Definition of Done

- Optional LLM judge exists with offline null provider.
- Manual script generates JSON and Markdown reports.
- Dashboard integration is informational only.
- CI does not depend on the judge.
- Documentation and Obsidian notes mark the feature as experimental.
