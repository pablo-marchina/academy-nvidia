# Final Delivery Index

This is the evaluator entrypoint for the benchmark-first final product.

## Reading Order

1. Read `README.md` for setup, runtime expectations, and limitations.
2. Read `docs/final_runtime_contract.md` for product-mode requirements.
3. Read `docs/final_benchmark_first_policy.md` for candidate adoption rules.
4. Read `docs/final_external_reviewer_mode.md` for clean-environment execution.
5. Read `docs/final_security_risk_plan.md`, `docs/final_ai_incident_response_plan.md`, and `docs/final_data_retention_policy.md` for operational controls.

## How To Run

```bash
cp .env.example .env
docker compose up -d
make setup
make prove-final-product
make run
```

## How To Validate Without Mock Or Demo Runtime

Run:

```bash
make validate-no-demo
python scripts/check_benchmark_type_policy.py
python scripts/check_runtime_value_policy.py
```

Then review:

- `final_case_evidence/no_demo_report.json`
- `final_case_evidence/mock_provider_benchmark_classification_report.json`
- `final_case_evidence/proxy_benchmark_promotion_block_report.json`

## How To Validate Benchmark-First Decisions

Review:

- `final_case_evidence/candidate_catalog.csv`
- `final_case_evidence/benchmark_results.jsonl`
- `final_case_evidence/output_value_benchmark_report.json`
- `final_case_evidence/decision_ledger.csv`
- `final_case_evidence/calibration_registry.csv`

Only `OUTPUT_VALUE` or `PRODUCTION_QUALITY` benchmark types may promote runtime adoption. `LOCAL_READINESS` and `PROXY` never promote alone.

## How To Audit A Recommendation

Trace from the action brief to:

- persisted startup and analysis run records
- claims and evidence coverage
- RAG support records
- NVIDIA mapping evidence
- confidence, uncertainty, business impact, implementation complexity, and next best action

Use `final_case_evidence/evidence_to_decision_coverage.json` and `final_case_evidence/data_lineage_report.json`.

## How To Reproduce The Release

The final delivery artifact is not a workspace ZIP. It is generated by:

```bash
make package-final-release
```

The expected artifact is `release/academy-nvidia-final-case.zip`.

`FINAL_PRODUCT_STATUS=PASS` means the configured environment passed the proof
gates. It does not mean every documented candidate was promoted to runtime.
Promotion remains evidence-based and requires benchmark records.
