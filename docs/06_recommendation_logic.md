# Recommendation Logic

## Principle

No recommendation is valid unless there is an explicit technical gap and evidence supporting that gap or a clearly marked inference.

## Flow

1. Review validated evidence.
2. Diagnose one or more technical gaps.
3. Map each gap to NVIDIA technologies.
4. Filter by technical fit and business usefulness.
5. Produce priority, complexity, and next action.

## Output quality checks

- Gap is explicit.
- Technology mapping is plausible.
- Technical justification is concrete.
- Business justification is relevant to NVIDIA.
- Evidence used is listed.
